Engine Overview

The top-level engine lifecycle, error propagation system, and conventions that every module follows. Start here before diving into the renderer or window subsystems.

// conventions

Coding Conventions

All functions across every module follow a consistent interface pattern.

Function Interface Rules

Return typeclz::Result — clz::Success() or Failure("location: msg"). Top-level lifecycle fns return bool.
Inputsconst references — read-only, never modified
OutputsRaw pointers prefixed with p — e.g. pInstance, pSwapchain
Partial failureFunctions do not clean up on partial failure — callers handle cleanup on bail-out
Error propagationCHECK for bool-returning fns · INVALIDATE for Result-returning fns
Global stateNone exposed — all functions operate on explicitly passed handles
// architecture

Engine Lifecycle

The engine has three phases driven from main.cpp via three calls into base.hpp. Each phase coordinates the window and renderer subsystems in the correct order.

① Startup — clz::init()
1window::init() — GLFW + window handle
2render::init() — full Vulkan stack
3Returns false on any failure via CHECK
② Main Loop — clz::update()
↺Runs until windowShouldClose()
1window::update(dt) — poll events
2render::update(dt) — render frame
③ Shutdown — clz::cleanup()
1render::cleanup() — Vulkan teardown
2window::cleanup() — GLFW teardown
✓Reverse of init order always
ℹ
Delta time: deltaTime passed to both window::update() and render::update() is currently hardcoded to 0 — reserved for future animation and input use.

Engine Lifecycle Functions

The three public functions in namespace clz that drive the entire engine. Called directly from main.cpp.

bool clz::init()
+
Initializes all internal engine subsystems. Calls window::init() first, then render::init(). Uses the CHECK macro to bail out early if either step fails.
Init Sequence
  • 1window::init() — GLFW init + 800×600 window creation
  • 2render::init() — full Vulkan stack (instance → swapchain → pipeline → sync → command pool)
  • 3Prints "Initialized all internal engines\nWelcome to curloz engine" on success
Returns
true if all subsystems came up cleanly · false on any failure
void clz::update()
+
Runs the main engine loop until the window signals a close. Pumps window::update(0) then render::update(0) every frame. Blocks until window::windowShouldClose() returns true.
Loop Body (per frame)
  • 1window::update(0) — poll OS events, check Escape key
  • 2render::update(0) — acquire image, record + submit commands, present
Returns
void — returns when windowShouldClose() is true
void clz::cleanup()
+
Tears down all engine subsystems in reverse initialization order. Shuts down the renderer before the window to avoid use-after-free of any window-backed rendering resources.
Teardown Order
  • 1render::cleanup() — vkDeviceWaitIdle then Vulkan object destruction
  • 2window::cleanup() — destroy GLFW window, terminate GLFW
// clz — include/clz_types.hpp

clz::Result

The engine's unified return type for all fallible operations. No exceptions — just a bool and a message.

struct clz::Result
bool success — true if operation succeeded
std::string message — human-readable status or error
void what() — prints message to stdout
clz::Success()
Returns a Result with success=true. Silent overload uses placeholder message. String overload logs to stdout.
clz::Failure(msg)
Returns a Result with success=false and the given message. Does not print — CHECK/INVALIDATE handle printing at the call site.

Error Propagation Macros

#define CHECK(result)
Used in bool-returning functions. Prints "ERROR: " + message and return false if result.success == false.
#define INVALIDATE(result)
Used in clz::Result-returning functions. Prints "ERROR: " + message and return result if result.success == false.

Public Renderer Interface

Three lifecycle functions mirroring the top-level engine pattern. All Vulkan state lives in module-level r_* variables in renderer.cpp.

ℹ
Frame state variables: r_currentFrame cycles 0..FRAMES_IN_FLIGHT-1. r_imageIndex is the swapchain image index from vkAcquireNextImageKHR — may differ from r_currentFrame. FRAMES_IN_FLIGHT = 2.
clz::Result clz::render::init()
+
Brings up the full Vulkan renderer stack. Any failure propagates immediately via INVALIDATE, leaving partial state uncleaned.
Init Sequence
  • 1initVulkan() — instance + validation layers + surface + device + queues
  • 2createSwapchain() — swapchain + image views
  • 3createShaderModules() — load SPIR-V from assets/shaders/
  • 4createPipeline() — graphics pipeline + layout
  • 5createSyncers() — semaphores + in-flight fences
  • 6createCommandPool() + createCommandBuffers()
Returns
Success or Failure if any subsystem fails
clz::Result clz::render::update(float deltaTime)
+
Renders one frame using the double-buffered frame-in-flight loop. All steps indexed by r_currentFrame. Advances r_currentFrame with modulo wrap at end.
Per-Frame Sequence
  • 1waitForGPU — block CPU until this frame's fence is signaled
  • 2acquireNextImage — get swapchain image index
  • 3startCommandBuffer — reset + begin
  • 4recordCommandBuffer — image barriers + draw call
  • 5submitCommandBuffer — vkQueueSubmit2
  • 6present — vkQueuePresentKHR
  • 7r_currentFrame = (r_currentFrame + 1) % FRAMES_IN_FLIGHT
Returns
Success or Failure if any Vulkan call fails
void clz::render::cleanup()
+
Tears down all Vulkan resources in reverse initialization order. Calls vkDeviceWaitIdle() first to guarantee the GPU is idle.
Destruction Order
  • 0vkDeviceWaitIdle()
  • 1command pool (frees all command buffers)
  • 2syncers (semaphores + fences)
  • 3pipeline + shader modules
  • 4swapchain + image views
  • 5device → surface → validation layers → instance

+
Records the full draw sequence using Vulkan 1.3 dynamic rendering — no render pass object.
Recording Sequence
  • 1Barrier: UNDEFINED → COLOR_ATTACHMENT_OPTIMAL
  • 2vkCmdBeginRendering — black clear, store op
  • 3Dynamic viewport + scissor to swapchain extent
  • 4Bind graphics pipeline
  • 5vkCmdDraw(3,1,0,0) — hardcoded triangle

// clz::render — include/renderer/cleaners.hpp

Cleaners

Counterpart destruction function for every initializer. renderer.cpp calls vkDeviceWaitIdle() before any of these. Destroy in reverse init order.

destroyCommandPool
Destroys pool and implicitly frees all allocated command buffers. No need to call vkFreeCommandBuffers first.
destroySyncers
Iterates all three vectors (imageAvailableSemaphores, renderCompleteSemaphores, inFlightFence) and destroys each handle.
destroyPipeline
Destroys graphics pipeline, pipeline layout, and both shader modules.
destroySwapchain
Destroys all image views first, then the swapchain. Images freed automatically by the swapchain.
destroyDevice
Destroys the logical device. Must be called after all device-owned resources are destroyed.
destroySurface
Must be called before destroyInstance().
destroyValidationLayers
Uses vkb::destroy_debug_utils_messenger. Safe if messenger is VK_NULL_HANDLE (release builds).
destroyInstance
Must be called last — after surface, validation layers, and device are already destroyed.
