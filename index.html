Coding Conventions

All functions follow a consistent interface pattern. Understanding it once makes the whole API predictable.

Function Interface Rules

Return typeclz::Result — either clz::Success() or Failure("location: message")
Input argumentsPassed as const references — read-only
Output argumentsRaw pointers prefixed with p — e.g. pInstance, pSwapchain
Partial failureFunctions do not clean up on partial failure — callers handle cleanup on bail-out
Namespacesclz::render for renderer · clz::window for windowing
Global stateNone — all functions operate on explicitly passed handles

The Render Loop

Each frame executes six steps in strict order, called by render::update() in renderer.cpp. These functions are not meant to be called directly by external code.

Mainloop Operations

Per-frame Vulkan render loop step implementations. All six functions operate on explicitly passed handles — no global state. Module-level state (r_currentFrame, r_imageIndex, etc.) lives in renderer.cpp.

clz::Result clz::render::waitForGPU(const types::Device device, VkFence *pFence)
+
Waits on the in-flight fence then resets it. Blocks indefinitely (UINT64_MAX timeout) until the GPU signals the fence, guaranteeing command buffer and sync primitives for this frame are free. Resets the fence for re-use by the next vkQueueSubmit2.

clz::Result clz::render::recordCommandBuffer(
    const types::CommandBuffer commandBuffer, const types::Swapchain swapchain,
    uint32_t imageIndex, types::Pipeline pipeline)
+
Records the full draw sequence for one frame using Vulkan 1.3 dynamic rendering — no render pass object.
Recording Sequence
  • 1Barrier: UNDEFINED → COLOR_ATTACHMENT_OPTIMAL
  • 2vkCmdBeginRendering — black clear, store op, single color attachment
  • 3Dynamic viewport + scissor set to swapchain extent
  • 4Bind graphics pipeline
  • 5vkCmdDraw(3, 1, 0, 0) — hardcoded triangle, vertex positions in shader
  • 6vkCmdEndRendering
  • 7Barrier: COLOR_ATTACHMENT_OPTIMAL → PRESENT_SRC_KHR
  • 8vkEndCommandBuffer
Parameters

commandBufferconst types::CommandBufferMust be in recording state
swapchainconst types::SwapchainSource of images, image views, and extent
imageIndexuint32_tWhich swapchain image to render into
pipelinetypes::PipelineGraphics pipeline to bind
Returns
Success or Failure if vkEndCommandBuffer fails

Window — Native (GLFW)

GLFW lifecycle implementation. All functions take an explicit types::Window* — no global state. The module-level w_window is owned by window.cpp and injected here via the forwarding layer.

⚠
Internal API: native.hpp is not the public engine API. Use window.hpp instead — it manages the module-level Window instance and forwards here.
clz::Result clz::window::init(types::Window *pWindow)
+
Initializes GLFW and creates a Vulkan-compatible 800×600 fixed-size window with no OpenGL context.
Sequence
  • 1glfwInit() — returns Failure on error
  • 2Hint GLFW_CLIENT_API = GLFW_NO_API (Vulkan, no GL context)
  • 3Hint GLFW_RESIZABLE = false (fixed 800×600)
  • 4glfwCreateWindow() — returns Failure if handle is null
Parameters

Returns
