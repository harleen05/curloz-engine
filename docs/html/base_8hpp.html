Detailed Description

Core engine lifecycle interface for Curloz Engine.

Exposes the three main lifecycle functions — init, update, and cleanup — that drive the engine's startup, main loop, and shutdown sequence.

◆ cleanup()

void clz::cleanup ( )

Tears down all engine subsystems in reverse initialization order.

Shuts down all internal engine subsystems in reverse init order.

Shuts down the renderer before the window to avoid use-after-free of any window-backed rendering resources. Prints shutdown messages to stdout for each step.

Cleans up the renderer first, then the window. Safe to call once after update() returns.

◆ init()

bool clz::init ( )

Initializes the window and renderer subsystems.

Initializes all internal engine subsystems.

Calls window::init() first, then render::init(). Uses the CHECK macro to bail out early if either step fails. Prints confirmation to stdout on success.

Returns
true if all subsystems came up cleanly, false on any failure.

Brings up the window and renderer in order. If any subsystem fails to initialize, the function returns false immediately.

Returns
true if all subsystems initialized successfully, false otherwise.

◆ update()

void clz::update ( )

Runs the engine's main loop.

Runs the main engine loop until the window signals a close.

Pumps window events and renderer updates every frame. Runs until window::windowShouldClose() signals that the window has been closed or a quit event was received.

Note
The delta-time argument passed to window::update() and render::update() is currently hardcoded to 0 (placeholder).

Ticks the window and renderer each frame in sequence. Blocks until window::windowShouldClose() returns true.
